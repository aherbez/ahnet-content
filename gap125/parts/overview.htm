Class Overview

Welcome! In this class, we will be covering the basics of the C++ programming language.

C++ is one of the most powerful and widely-used languages out there, and is used pretty much anywhere a computer is found

Each week, we'll cover a new set of topics releated to C++. Weekly assignments will be given in which you will have a chance to practice the concepts we discuss in class. I'll also be giving short quizzes to help keep previous topics fresh.

Makeup of your final grade

Your final grade will consist of the following:
Source % of Final
Quizzes 5%
Assignments 60%
Midterm 15%
Final 20%
More on each of the above follows below.

Quizzes

Most classes will begin with a short quiz covering previous topics. These will not be hard, and are meant mainly as a way to keep the content from previous classs fresh in your mind. When calculating your quiz score, I will discard two of your lowest scoring quizzes (one from the first half of the semester, and one from the second half).

Assignments

Each week, you will be presented with an assignment consisting of one or more programming challenges. You are to write code to complete the challenges, and submit the files to me no later than the start of the following class

Any submissions received later than the start of the next class (3:30pm, PST, the Friday following the one in which the assignment was first discussed), will be considered late, and will receive a penalty to the grade they earn.

Submitting assignments

Assignments should be placed in a folder, and that folder should be compressed to a single .ZIP file. The filename for the ZIP should include:

  • Your last name
  • The assignment number
So, if I was going to submit my work for week 1, I would email (myself) a zip file named "Herbez_week1.zip"

If you have reason to submit more than once, please include a further number in the file name. So, if I had realized I had forgotten a file in the above submission, I would name the next zip "Herbez_week1_2.zip"

The Midterm

The midterm will be similar to the assignments, but slightly larger in scope. I will be giving you a list of problems to solve, and expecting you to select a few from the list to implement.

The midterm will be submitted in the same way as the assignments- as a ZIP file emailed to me at aherbez@gmail.com

More on the midterm as we get closer

Final Projects

The class will end with you creating a project of your own choosing. Don't worry if you're not sure what to do just yet- I will be working with you as the semester progresses to help you pick a project that is both challenging enough for you to learn on, while not being too large to fit into a few weeks of work.

Plagarism

You MAY NOT:

  • Copy & Paste code you find on the internet for use in your projects.
  • Use code you download from the internet.
  • Use any code from another student.

You MAY:

  • Use code from YOUR OWN previous projects.
  • Look at code on the internet to get an idea of how to implement something.
  • Use code snippets from the module lessons and demos (though I would suggest making your own version)

It is never, ever okay to directly use & apply code from the internet or other students on assignments. At a real game company, that would likely be plagiarism (depending on the license) and could kill the company if the original author got wind of it.

If you have any questions or would like clarification, let me know.
